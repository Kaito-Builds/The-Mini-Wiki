# 🌐 The Mini-Wiki Project

A multi-page architectural sprint designed to master internal linking, asset management, and directory navigation.

## 🚀 The Goal
To transition from single-page layouts to a functional "Site Skeleton," ensuring seamless user flow between index and informational sub-pages.

## 🛠️ Features
- **Multi-Page Navigation:** Functional header with absolute and relative linking.
- **Asset Optimization:** Integrated images with proper `alt` text for SEO/A11y.
- **Internal Anchors:** Implementation of `#` jumps for long-form content navigation.
- **Clean File Paths:** Organized directory structure for high-scalability.

## 🧠 What I Learned
- The "Don't Break the Link" rule: Maintaining relative paths during site growth.
- How to structure an "About" page that supports the main brand.
- The importance of a consistent `<nav>` across different HTML files.
